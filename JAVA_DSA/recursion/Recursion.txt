    ‍‍⁡⁣⁢⁢Definition⁡ 
    
    - Recursion means a ⁡⁢⁣⁣function⁡ calling ⁡⁢⁣⁡⁢⁣⁣𝗶𝘁𝘀𝗲𝗹𝗳⁡.
    - ⁡⁢⁣⁣Base_condition⁡ - condition where recursion will stop making new calls.( generally a simple ⁡⁢⁣⁣if-condition⁡).
    - No base-condition -> function calls will keep happening , stack will be filled again and again < if no base condition is given the function will keep on calling it self -> then the stack will be filled > - ⁡⁢⁣⁣Memory of the computer will exceed the limit⁡ - > ⁡⁣⁣⁢Stack-Overflow-Error⁡
    - If you call a function again and again , you can treat each as a seperate function call in a stack.
    
    ⁡⁣⁢⁢Why Recursion⁡ 
    
    - It helps us in solving bigger / complex problems in a simpler way by breaking it down in smaller more manageable problems.
    - You can convert recursion solution into iterative solution and vice-versa.
    - Space complexity is not constant because of recursive calls.

    ⁡⁣⁢⁢How solve any Problem using Recursion -⁡

    Steps-
    - Identify if you can break down the problem into samller problems.
    - Write the Recurrence relation if needed.
    - Draw the recursive tree.
    - About the tree:
        -See the flow of functions, how they are getting in stack.
        - Identify & focus on left tree calls and then right.
        - Draw the tree and pointer again again using pen and paper.
    - See how the values are returned at each step, see where the function call will come out , In the end , uou will come out of the main.
